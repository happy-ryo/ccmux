# Unix IPC ソケットディレクトリの UID 解決を環境変数依存から外す

## 背景

Unix 側 IPC エンドポイントは `$XDG_RUNTIME_DIR/ccmux/` を優先し、無い場合は `/tmp/ccmux-<uid>/` を使う実装になっている。

ただし現在の `uid()` は実 UID ではなく `$UID` 環境変数に依存しており、未設定時は `1000` にフォールバックしている。環境によっては別ユーザーと同じディレクトリ名に衝突し得るうえ、コメントで前提にしている owner-only directory の保証が実装とずれている。

## 対象コード

- `src/ipc/endpoint.rs`

## 対応内容

- Unix では実 UID を OS から取得する実装に切り替える
- `$UID` 文字列や固定値 `1000` へのフォールバックをやめる
- IPC ディレクトリ作成後、owner-only permission を設定できなかった場合の扱いを見直す
- 必要なら permission 不備を soft-fail ではなく明示エラーにする
- テストを追加して `XDG_RUNTIME_DIR` なし環境でも UID ごとにディレクトリ名が安定することを確認する

## 受け入れ条件

- Unix で実行ユーザーごとに一意な IPC ディレクトリ名が使われる
- `$UID` 未設定でも `1000` 固定に落ちない
- owner-only directory 前提がコードとテストで担保される
- 既存の Windows 動作には影響しない

## 補足

影響は主にクロスユーザー衝突や DoS 寄りだが、セキュリティ説明と実装を一致させるために先に潰しておきたい。
